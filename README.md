# Face Recognition Portal (FaceNet-based)

A face recognition system that learns to identify people from photos, then
recognizes them via live webcam or uploaded photos through a Streamlit web
interface.

---

## How it works

The system uses **FaceNet** (an `InceptionResnetV1` model pretrained on
VGGFace2) instead of classical OpenCV/LBPH recognition:

1. **MTCNN** detects and aligns faces in an image — far more robust across
   angles, lighting, and poses than Haar cascades.
2. **InceptionResnetV1** converts each aligned face into a **512-dimensional
   embedding vector** (a numeric "fingerprint" of the face). Embeddings are
   L2-normalized.
3. **Recognition = nearest-neighbor search.** A new face's embedding is
   compared (via Euclidean distance) against every embedding stored in the
   "gallery." If the closest match is within `FACE_DISTANCE_THRESHOLD`, that
   person's name is returned; otherwise the face is labeled `Unknown`.

There is no traditional "training" step that fits a classifier — building
the model just means computing and storing embeddings for your reference
images. This makes the system robust to uneven amounts of data per person
(e.g. 13 images for one person, 300 for another), since each embedding is
simply an extra reference point rather than something that biases a decision
boundary.

## Project files

```
project/
├── face_engine.py        Shared FaceNet detection + embedding utilities
├── collect.py             CLI: import datasets / capture webcam samples
├── train.py               CLI: build/update the embedding gallery
├── app.py                 Streamlit web app (recognition only)
├── face_data/             Your training images, organized by subject
│   ├── Alice/
│   │   ├── alice_0001.png
│   │   └── ...
│   └── Bob/
│       └── ...
└── face_embeddings.npz    The embedding gallery (generated by train.py)
```

### `face_engine.py`
Shared module used by both `train.py` and `app.py`. Loads MTCNN (face
detector) and InceptionResnetV1 (embedding model) once. Key functions:

- `detect_faces(pil_img)` — returns `[(box, prob), ...]` for every face found
  above a confidence threshold (`DETECTION_PROB_THRESHOLD = 0.90`).
- `embed_face(pil_img, box)` — crops/aligns the face at `box` and returns its
  512-d embedding.
- `embed_largest_face(pil_img)` — convenience for training: embeds the
  largest face found in an image.

Runs on GPU automatically if CUDA is available (`DEVICE` auto-detects),
otherwise CPU.

### `collect.py`
Builds up `face_data/<subject_name>/` folders of images. Two modes:

- **`python collect.py zip <file.zip>`** — extracts a dataset zip. Handles
  nested folder structures like `actors/personname/image.jpg`; the **direct
  parent folder name** becomes the subject's identity. Images are copied into
  `face_data/<personname>/`.
- **`python collect.py webcam "<Name>" --count 40 [--camera 0]`** — opens your
  webcam, shows a live preview with a face-detection box (for feedback only),
  and saves a full frame each time you press SPACE (ESC to stop early).

By default, **both modes automatically trigger an incremental gallery update**
afterward (see `train.py`). Add `--no-train` to skip this and run
`python train.py` manually later.

### `train.py`
Builds/updates `face_embeddings.npz`, the embedding gallery.

- **Incremental by default.** The gallery file stores a `processed` list of
  every image file (`subject_name/filename`) already embedded. Re-running
  only embeds *new* files — already-processed images are skipped entirely.
  This is what makes adding one new person fast, even with a gallery of
  thousands of images.
- **Checkpointed.** After every subject finishes (even if interrupted
  mid-subject with Ctrl+C), the gallery is saved immediately. Safe to stop
  and resume at any time — just re-run `python train.py`.
- **`--full`** — ignores the existing gallery and recomputes every embedding
  from scratch (useful if you changed `DETECTION_PROB_THRESHOLD` or other
  embedding-affecting settings in `face_engine.py`).
- **Threshold tuning.** After updating embeddings, it holds out ~30% of each
  subject's embeddings as test queries, runs 1-NN against the rest, and
  sweeps distance thresholds from 0.40 to 1.50 to find the one that maximizes
  recognition accuracy. It prints a table and a **recommended
  `FACE_DISTANCE_THRESHOLD`** — copy this value into `app.py`.

`face_embeddings.npz` contains:
| key | contents |
|---|---|
| `embeddings` | `(N, 512)` float32 array of face embeddings |
| `labels` | `(N,)` int32 array — which subject each embedding belongs to |
| `mapping` | label ID → subject name |
| `processed` | set of `subject/filename` strings already embedded |

### `app.py`
The Streamlit web interface, with exactly two tabs:

- **📷 Live Webcam Recognition** — uses `streamlit-webrtc` to stream your
  webcam, drawing a bounding box + name + distance score over every detected
  face in real time. To keep this responsive on CPU, recognition only re-runs
  every `PROCESS_EVERY_N_FRAMES` frames (default `5`); intermediate frames
  reuse the last result. Lower this if you have a GPU.
- **🖼️ Photo Recognition** — upload any image; it detects all faces, labels
  each with its closest match and distance score, and lists the results
  below the image.

Configuration at the top of the file:
```python
FACE_DISTANCE_THRESHOLD = 0.90   # update from train.py's recommendation
PROCESS_EVERY_N_FRAMES = 5       # live recognition frame skip
```

The sidebar shows whether the gallery loaded successfully, how many subjects
are enrolled, and the total embedding count.

## Setup

```bash
# Core dependencies
pip install streamlit streamlit-webrtc av opencv-contrib-python numpy Pillow

# PyTorch (CPU build -- avoids pulling the huge CUDA wheel unless you have a GPU)
pip install torch --index-url https://download.pytorch.org/whl/cpu

# FaceNet
pip install facenet-pytorch
```

The first time `train.py` or `app.py` runs, `facenet-pytorch` downloads the
pretrained InceptionResnetV1 weights (~110MB) to `~/.cache/torch` — this
needs an internet connection once, then works offline.

## Typical workflow

```bash
# 1. Import dataset(s) -- handles nested actor/person/image.jpg structures
python collect.py zip dataset1.zip
python collect.py zip dataset2.zip

# 2. Add yourself or anyone else via webcam (30-50+ varied images recommended)
python collect.py webcam "Your Name" --count 40

# Each of the above automatically rebuilds the gallery incrementally.
# If you used --no-train on any of them, run this once at the end:
python train.py

# 3. Copy the "Recommended FACE_DISTANCE_THRESHOLD" value train.py printed
#    into FACE_DISTANCE_THRESHOLD in app.py

# 4. Run the app
streamlit run app.py
```

### Adding a new person later
```bash
python collect.py webcam "New Person" --count 40
```
This embeds only the new images and appends them to the existing gallery —
nothing else gets re-processed.

## Tuning & troubleshooting

- **Too many faces classified as "Unknown"** → raise
  `FACE_DISTANCE_THRESHOLD` slightly (more lenient matching).
- **Wrong person identified too often** → lower `FACE_DISTANCE_THRESHOLD`
  (stricter matching) — re-run `python train.py` after changing it if you
  want the threshold sweep re-evaluated, though the sweep itself doesn't
  depend on this setting.
- **"no usable faces detected" for a subject** → MTCNN couldn't find a face
  above `DETECTION_PROB_THRESHOLD = 0.90` in any of that subject's images.
  Try lowering it to ~0.85 in `face_engine.py`, then `python train.py --full`.
- **Live webcam feels laggy** → increase `PROCESS_EVERY_N_FRAMES` (less
  frequent recognition) or, if you have an NVIDIA GPU, install the CUDA
  build of PyTorch instead of the CPU build — `face_engine.py` will use it
  automatically.
- **Training a huge dataset takes hours** → expected on CPU for thousands of
  images. It's safe to interrupt (Ctrl+C) at any point — progress is
  checkpointed after each subject, so re-running `python train.py` resumes
  where it left off.
- **A subject has very few images (under ~5)** → `train.py` will warn about
  this; recognition for that person will likely be less reliable. Add more
  images via `python collect.py webcam "<Name>" --count 20`.

## Recognition output meaning

Both the live feed and photo results show a **distance score** next to each
name (e.g. `Alice (0.52)`). This is the Euclidean distance between the
detected face's embedding and its closest gallery match:

- **Lower = more confident match** (0 = identical embeddings)
- **Higher = less confident** — if it exceeds `FACE_DISTANCE_THRESHOLD`, the
  face is labeled `Unknown` instead
